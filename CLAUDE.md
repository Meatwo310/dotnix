# CLAUDE.md — dotnix

This file provides guidance for AI assistants working in this repository.

## Overview

**dotnix** is a NixOS Flake-based system configuration repository managing two machines declaratively:

- **sp9-v7** — Microsoft Surface Pro 9 (native NixOS, KDE Plasma 6 desktop)
- **gaming-wsl** — Windows Subsystem for Linux gaming machine (NixOS-WSL)

The primary user account across both hosts is `moon`.

## Repository Structure

```
flake.nix                        # Flake entry point: inputs and nixosConfigurations outputs
flake.lock                       # Pinned dependency versions
modules/
  common.nix                     # Universal baseline (Nix settings, locale, user, nh, vim, git)
  extra.nix                      # Extended services (NetworkManager, SSH, Tailscale, VSCode Server)
  desktop.nix                    # Desktop stack (X11, KDE Plasma 6, ly, PipeWire, Firefox)
hosts/
  sp9-v7/
    configuration.nix            # Boot loader, hostname, keyboard layout
    hardware-configuration.nix   # Auto-generated hardware config (do not edit manually)
  gaming-wsl/
    configuration.nix            # WSL enable, defaultUser, hostname
packages/
  linux-surface.nix              # Custom Surface kernel package definitions (for CI)
.github/workflows/
  build.yml                      # CI: flake check → build-heavy → build hosts
  pinact.yml                     # CI: verify GitHub Actions are pinned to commit hashes
.vscode/
  settings.json                  # nil LSP configuration
  extensions.json                # Recommended: jnoortheen.nix-ide
```

## Module Composition

Hosts import modules additively. Each module has a single responsibility:

| Module | Responsibility |
|--------|---------------|
| `modules/common.nix` | Nix settings, Cachix caches, locale (ja_JP/en_US), timezone (Asia/Tokyo), `nh` package manager, user `moon`, vim, git, wget |
| `modules/extra.nix` | NetworkManager, OpenSSH, Tailscale, VSCode Server, nil LSP, nixpkgs-fmt |
| `modules/desktop.nix` | X11, ly login manager, KDE Plasma 6, PipeWire audio, printing, Firefox, Kate |

**sp9-v7** module load order (matters for NixOS option merging):
1. `nixos-hardware.nixosModules.microsoft-surface-pro-9`
2. `vscode-server.nixosModules.default`
3. `./modules/common.nix`
4. `./modules/extra.nix`
5. `./modules/desktop.nix`
6. `./hosts/sp9-v7/configuration.nix`

**gaming-wsl** module load order:
1. `nixos-wsl.nixosModules.default`
2. `./modules/common.nix`
3. `./hosts/gaming-wsl/configuration.nix`

## Common Development Commands

```bash
# Check flake syntax and evaluation
nix flake check

# Build a host system closure (dry-run, no activation)
nix build .#nixosConfigurations.sp9-v7.config.system.build.toplevel --no-link
nix build .#nixosConfigurations.gaming-wsl.config.system.build.toplevel --no-link

# Build custom Surface kernel package
nix build .#packages.x86_64-linux.linux-surface-lts --no-link

# Apply configuration on the target host (using nh)
nh os switch

# Format Nix files
nixpkgs-fmt <file.nix>
```

## Coding Conventions

- **Formatter**: `nixpkgs-fmt` — run before committing Nix files
- **Module signature**: use `{ config, pkgs, ... }:` (include only what the module needs)
- **Comments**: Japanese and English are both used; prefer Japanese for end-user-facing context, English for technical NixOS attribute names
- **Comment style**: `# comment` (line comments only; no block comments)
- **Attribute sets**: use standard NixOS formatting — 2-space indentation, braces on same line
- **Hardware config**: `hardware-configuration.nix` is auto-generated by `nixos-generate-config`; do not edit manually unless necessary
- **Adding a new host**: create `hosts/<hostname>/configuration.nix`, add a new entry in `flake.nix` `nixosConfigurations`, and update the CI matrix in `.github/workflows/build.yml`
- **Adding a new module**: place in `modules/`, import it explicitly in relevant host entries in `flake.nix`
- **No home-manager**: this repo uses only NixOS system modules, not home-manager

## CI/CD Pipeline

Three sequential jobs defined in `.github/workflows/build.yml`:

1. **check** — runs `nix flake check` on every push/PR
2. **build-heavy** (needs: check) — builds `linux-surface-lts` kernel; pushes to Cachix; runs weekly on schedule (Mon 0:00 UTC) for cache freshness
3. **build** (needs: build-heavy) — builds both `sp9-v7` and `gaming-wsl` host closures in parallel via matrix strategy

All GitHub Actions versions must be pinned to full commit hashes (enforced by `pinact.yml`). When adding or updating Actions, always use the `owner/repo@<commit-hash> # vX.Y.Z` format.

Required secrets:
- `CACHIX_AUTH_TOKEN` — for pushing to `meatwo310-dotnix` Cachix cache
- `GITHUB_TOKEN` — automatically provided by GitHub Actions

## What Not to Do

- Do not edit `hosts/sp9-v7/hardware-configuration.nix` unless regenerating from hardware
- Do not add packages to `modules/common.nix` that are host- or desktop-specific; use `extra.nix` or `desktop.nix`
- Do not use `nix-env` or imperative package installs; all packages must be declared in Nix modules
- Do not skip `nix flake check` before committing — CI will catch evaluation errors
- Do not unpin GitHub Actions versions (pinact.yml enforces pinning)
